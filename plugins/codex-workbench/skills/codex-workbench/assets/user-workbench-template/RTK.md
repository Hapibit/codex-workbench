# RTK 使用规则

RTK 用于压缩低信号 shell 输出，不是所有命令的默认包装器。

## 适合使用

- 输出很长但只需要摘要。
- 大量日志、测试输出、构建输出需要提取关键错误。
- 需要减少重复噪声，保留失败原因和文件位置。

## 不适合使用

- 需要精确原始输出的命令。
- 需要交互的命令。
- 输出本身很短的命令。
- 会因为包装而改变退出码、路径、编码或交互行为的命令。

## 规则

- 先保证命令本身可运行，再考虑压缩输出。
- 失败命令要保留退出码、关键错误、文件路径和行号。
- 中文规则、Markdown 和 PowerShell 输出优先按 UTF-8 读取，避免乱码误判。
