# ADR 目录

本目录保存 Architecture Decision Record。每个重大架构、数据、API、AI、部署或安全决策都应该有一条 ADR。

文件名建议：

```text
0001-short-title.md
```

## 模板

```text
# ADR-0001 标题

Status: proposed|accepted|superseded
Date: YYYY-MM-DD

## Context

为什么需要做这个决策，约束是什么，哪些需求或风险触发了它。

## Decision

最终选择是什么。

## Alternatives

考虑过哪些替代方案，为什么没有选。

## Consequences

好处、代价、风险、回滚方式、后续验证。
```
