## 规则

- 单个功能的细节证据先放在功能包 `CHANGELOG.md`、`VERIFY.md`、`REVIEW.md`。
- 跨功能或影响工作台流程的问题再汇总到这里。
- 如果同类问题重复出现，要升级模板、测试、质量门、CI、hook 或审查清单。
