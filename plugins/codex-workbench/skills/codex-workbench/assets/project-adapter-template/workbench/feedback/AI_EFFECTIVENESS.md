# AI 效果评估

本文件用于评估 AI 实现、审查和修改后的实际效果，帮助工作台持续改进。

## 指标

| 指标 | 记录方式 | 目标 |
| --- | --- | --- |
| 一次通过率 | 功能包 VERIFY/REVIEW 是否首次通过 | 趋势改善 |
| 返工次数 | CHANGELOG 和 REVIEW 中的返工记录 | 下降 |
| 审查发现数 | P0/P1/P2 数量 | 高危问题前移 |
| 质量门失败数 | quality_gate 失败记录 | 同类失败减少 |
| 需求澄清次数 | CLARIFY 阻塞问题 | 模板逐步补强 |

## 规则

- AI 做错不是只改代码；要判断是需求、设计、计划、测试、审查还是质量门的问题。
- 如果修改后效果仍不好，记录原因并升级工作台，而不是继续重复同一提示。
